# 文档系统规范

这个文件定义 `omne_foundation` 在 agent-first 语境下如何组织仓库内文档。

目标不是再造一个巨大的“总说明书”，而是把文档维护成一张稳定地图，让人和 agent 都能逐步下钻到更窄、更具体的事实来源。

## 入口分层

根入口固定分成五层：

1. `AGENTS.md`
2. `ARCHITECTURE.md`
3. `docs/README.md`
4. `docs/规范/README.md` 与 `docs/crates/README.md`
5. 更窄的 `docs/规范/<topic>.md`、`crates/<crate>/README.md` 与各 crate 自身专题文档

这些入口各自负责不同粒度的问题，不能混成一个大文件。

## 每层负责什么

### `AGENTS.md`

- 只保留根入口、硬边界和最低检查命令
- 不重复 crate 细节
- 不承载实现级说明

### `ARCHITECTURE.md`

- 只保留 workspace 顶层分层
- 只说明 crate 之间的主要依赖方向
- 只给出记录系统入口，不展开主题细节

### `docs/README.md`

- 作为 workspace 级文档地图
- 提供稳定入口与目录约定
- 明确文档的渐进式披露规则

### `docs/规范/`

- 只承载 workspace 级治理规则
- 每个文件只覆盖一个稳定主题
- 文档负责解释边界、目的、例外和人工判断点，不重复脚本实现

### `docs/crates/` 与 crate 自身文档

- `docs/crates/README.md` 负责 crate 索引
- `crates/<crate>/README.md` 负责 crate 级领域、边界、范围、结构设计
- crate 自身 `docs/` 负责更窄的专题细节

## 维护规则

- 根级入口文档必须保持短小，优先做地图。
- 新增长期规则时，优先放到 `docs/规范/`，不要把根入口越写越厚。
- 新增 crate 事实时，优先写到对应 `crates/<crate>/README.md`。
- 能机械检查的结构约束，优先接入 `scripts/workspace_check/`。

## 当前机械约束

文档系统当前至少要求：

- 根级 `README.md` 必须存在
- 根级 `AGENTS.md` 必须存在
- `docs/docs-system-map.md` 必须存在
- `AGENTS.md` 必须保持短小
- `README.md`、`AGENTS.md`、`docs/README.md`、`docs/docs-system-map.md` 必须保留关键入口之间的互相指向
- `scripts/check-workspace.sh` 的 `local` / `ci` 模式都会先跑文档系统检查

也可以单独执行：

```bash
scripts/check-workspace.sh docs-system
```

## 与其他规范的关系

- 文档系统约束先定义“地图怎么组织”
- `版本与兼容.md`、`提交与分支.md`、`变更记录.md`、`Hook与质量门禁.md` 再定义各自主题的规则
- 这些规范文件都属于记录系统的一部分，而不是旁注
